# FitLife - Fitness App

FitLife is a Flutter + Firebase fitness tracking app for workout planning, logging, and progress tracking.

## 🚀 What This Project Does

- 🔐 User authentication with Email/Password and Google Sign-In
- 🏋️ Workout categories (Strength, Cardio, Core, Stretching)
- ⏱️ Exercise detail screens with timer and set/reps flow
- 📈 Progress tracking from Firestore workout logs
- 👤 Profile management with editable user details and avatar
- 🌗 Light/Dark theme with local persistence
- 🆘 Help & Support and About pages

## 🧰 Tech Stack

- Flutter (Dart SDK `^3.11.0`)
- Firebase Core, Firebase Auth, Cloud Firestore
- Provider (theme management)
- Shared Preferences (local settings/session helpers)
- Google Sign-In
- Image Picker, Path Provider, URL Launcher

## 📁 Project Structure

```text
lib/
  main.dart                   # App startup + Firebase initialization
  firebase_options.dart       # Generated by FlutterFire
  exercise/                   # Exercise pages and detail flow
  pages/                      # Auth, home, profile, settings, progress, etc.
  services/                   # Firestore + local preference helpers
images/                       # Local assets
android/ ios/ web/ windows/   # Platform folders
```

## ✅ Prerequisites

Before running the app, make sure you have:

- Flutter SDK installed and available in `PATH`
- A Dart SDK compatible with `^3.11.0`
- A Firebase project configured for your target platform(s)

Check your setup:

```bash
flutter --version
```

## 🔥 Firebase Setup

1. Create a Firebase project.
2. Enable Authentication providers:
 - Email/Password
 - Google
3. Create a Firestore database.
4. Register your Android/iOS/Web apps in Firebase.
5. Add platform config files:
 - `android/app/google-services.json`
 - `ios/Runner/GoogleService-Info.plist`
6. Generate or refresh Firebase options:

```bash
flutterfire configure
```

## 🛠️ Troubleshooting

### 1) Firebase initialization fails

- Confirm `lib/firebase_options.dart` exists and matches your Firebase project.
- Confirm Android/iOS config files are correct.

### 2) Google Sign-In fails on Android

- Add SHA-1 and SHA-256 certificates in Firebase Console.
- Download the latest `google-services.json`.

### 3) Google Sign-In fails on Web

- Ensure Google provider is enabled in Firebase Authentication.
- Ensure your Firebase authorized domains include your web host.
- If you customize sign-in behavior with `google_sign_in_web`, set proper web client configuration.

### 4) Firestore permission errors

- Check Firestore rules and confirm the signed-in user can access their own docs.
